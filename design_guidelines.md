# 人力资源管理系统 - 设计指南

## 品牌定位
- 应用类型：企业人力资源管理系统
- 设计风格：专业、简洁、高效
- 目标用户：新员工、企业HR
- 设计理念：简洁清晰的界面，快速高效的流程

## 配色方案

### 主色板
- **主色调（Primary）**: 蓝色系，体现专业和信任感
  - `bg-blue-600` / `text-blue-600` - 主要按钮、高亮
  - `bg-blue-50` / `text-blue-700` - 次要背景
  - `hover:bg-blue-700` - 按钮悬停

### 中性色
- **背景色**
  - `bg-white` - 主背景
  - `bg-gray-50` - 分组卡片背景
  - `bg-gray-100` - 列表项背景

- **文字色**
  - `text-gray-900` - 主要文字
  - `text-gray-600` - 次要文字
  - `text-gray-400` - 辅助文字

### 语义色
- **成功**: `text-green-600` / `bg-green-50`
- **警告**: `text-yellow-600` / `bg-yellow-50`
- **错误**: `text-red-600` / `bg-red-50`
- **信息**: `text-blue-600` / `bg-blue-50`

## 字体规范
- **页面标题**: `text-xl font-bold text-gray-900`
- **卡片标题**: `text-lg font-semibold text-gray-900`
- **正文字体**: `text-sm text-gray-600`
- **辅助文字**: `text-xs text-gray-400`
- **按钮文字**: `text-sm font-medium`

## 间距系统
- **页面边距**: `px-4 py-4`
- **卡片间距**: `gap-4`
- **卡片内边距**: `p-4`
- **列表项间距**: `gap-3`
- **元素间距**: `gap-2`

## 组件使用原则

### 通用组件优先级
**所有通用 UI 组件优先使用 `@/components/ui/*`，禁止手搓**

#### 按钮类
- **主要操作**: `Button` - 提交、确认、上传
- **次要操作**: `Button variant="outline"` - 取消、返回
- **危险操作**: `Button variant="destructive"` - 删除

#### 输入类
- **单行输入**: `Input` - 姓名、手机号
- **多行输入**: `Textarea` - 备注
- **表单标签**: `Label` - 字段说明
- **表单项**: `Field` - 统一表单结构

#### 布局类
- **卡片容器**: `Card` + `CardContent` - 信息分组
- **列表容器**: `Card` - 列表项
- **分割线**: `Separator` - 内容分隔

#### 提示类
- **页面内提示**: `Alert` - 重要提醒
- **操作反馈**: `Toast` / `Sonner` - 操作成功/失败
- **加载态**: `Skeleton` - 数据加载中

#### 其他
- **状态标签**: `Badge` - 已提交、待审核
- **弹窗**: `Dialog` - 确认提示
- **进度条**: `Progress` - 上传进度

## 导航结构

### 小程序端（新员工）
- **主页面**: 资料上传页面
  - 基本信息填写
  - 身份证上传（2张）
  - 学位证书上传（4张）
  - 体检报告上传（数量不限）
  - 离职证明上传（1张）
  - 提交按钮

### Web端（HR后台）
- **登录页面**: 管理员登录
- **主页面**:
  - 顶部导航：搜索、筛选
  - 员工列表
  - 员工详情（查看资料、下载）

## 页面容器规范

### 卡片样式
- **背景**: `bg-white`
- **圆角**: `rounded-xl`
- **内边距**: `p-4`
- **阴影**: `shadow-sm`
- **边距**: `mb-4`

### 列表项样式
- **背景**: `bg-white`
- **圆角**: `rounded-lg`
- **内边距**: `p-3`
- **边距**: `mb-2`
- **阴影**: `shadow-none`
- **边框**: `border border-gray-200`

## 小程序约束
- **包体积限制**: 图片通过对象存储，不打包到项目
- **图片策略**: 所有图片上传到TOS对象存储，代码中使用URL引用
- **性能优化**: 懒加载列表，分页加载数据
- **跨端兼容**: 同时支持微信小程序和H5

## 特殊说明
1. **文件上传**: 使用 `Network.uploadFile()`，禁止硬编码 localhost
2. **跨端兼容**: Taro 原生组件需要平台检测，H5端提供降级方案
3. **数据验证**: 前端验证文件格式和数量，后端二次验证
4. **错误提示**: 使用 Toast 提示用户操作结果
5. **加载状态**: 上传过程显示进度或加载态

## 状态展示
- **空状态**: 使用 Alert 提示用户开始操作
- **加载态**: 使用 Skeleton 显示占位
- **错误态**: 使用 Alert（variant="destructive"）提示错误
- **成功态**: 使用 Toast 或 Sonner 提示成功
